The shift

Today vs what we're proposing

Same engine underneath. Completely different feel.

Today

List editing

Stacked field cardsFeels like configuring settings, not designing an intake.
Preview hidden in AdvancedOnly appears below the builder — surprise on publish.
Logic in a popoverOperators, field keys — technical, not human.
Reorder via arrowsMarketing says drag-and-drop — grip icon only.
Save & publish onlyNo autosave — consultants hesitate to experiment.
Proposed

Interactive canvas

Split-pane workspaceBuild left, live client preview right — always on.
Real drag-and-dropFields + logic blocks reorder naturally.
Plain-English rules"When visa is 482 → show Employer name"
Logic as visible blocksPurple cards between questions — not buried.
Autosave draftsExperiment freely. Publish when ready.
Why revamp

7 things holding the builder back

Not form designing

Consultants think in scenarios, not field rows.

Preview disconnected

Can't see client view while editing logic.

Logic too technical

equals / is_one_of / field keys scare people.

No real drag

Arrows + icon ≠ the promised experience.

No sections

Long forms feel like one flat scroll.

Scary save model

One button to publish — no safe drafts.

Gideon's ask

"Drag and drop, like Legal Fun — user-friendly for complex forms."
